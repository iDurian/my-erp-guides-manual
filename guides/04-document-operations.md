# 通用单据操作

本模块用于讲解所有业务单据共通的操作方式和状态规则。

## 常用操作

所有业务单据都应覆盖以下操作：

- 新增单据。
- 保存草稿。
- 提交审核。
- 审核通过。
- 退回草稿。
- 作废单据。
- 从上游单据下推生成下游单据。
- 查看来源单据和上下游关系。
- PDF 预览和导出。
- Excel 导出。
- 附件上传、预览、下载和删除。

## 单据状态

常见状态流转：

```text
草稿 -> 待审核 -> 已审核 / 已过账 -> 执行中 -> 已完成
```

作废是例外操作，需要特别谨慎。

## 编辑规则

- 普通业务单据通常只有草稿状态可编辑。
- 待审核单据需要先退回草稿才能修改。
- 已审核或已过账单据修改会受限制。
- 已有有效下游单据的上游单据不能随意作废。

## 下推规则

- 优先使用下推生成下游单据。
- 下推可以减少重复录入。
- 下推可以保留来源和追溯关系。
- 手工新建下游单据时，应确认关联合同和来源信息是否正确。

## 导出和附件

- 非客户询价单据通常支持 PDF 预览和导出。
- 报价单、客户合同或 PI 类文件需要重点检查客户信息、产品明细、贸易条款、收款账户和公司资料。
- 合同类单据可维护附件，便于后续归档和查证。

## 讲解建议

选一张单据演示完整状态变化：保存草稿、提交审核、审核通过、下推下游单据，再说明什么情况下不能编辑和作废。
