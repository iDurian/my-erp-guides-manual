# 核心业务流程

本模块用于讲解系统最重要的一条业务主线。建议用一张销售合同贯穿演示完整链路。

## 主流程

```text
报价单
  -> 销售合同
  -> 申购单
  -> 采购合同
  -> 提货通知单
  -> 采购入库单
  -> 库存出库单
  -> 销售发票 / 采购发票
  -> 收款单 / 付款单
```

## 各环节说明

### 报价单

- 登记客户报价。
- 选择客户和产品。
- 形成对外价格、数量和贸易条款。
- 客户确认后可下推销售合同。

### 销售合同

- 是后续申购、采购、出库、收款和归档的源头。
- 客户确认后形成合同。
- 后续业务应尽量从销售合同下推，保证追溯关系完整。

### 申购单

- 由销售合同下推。
- 读取当前可用库存。
- 锁定可用库存。
- 计算需要额外采购的缺口数量。

### 采购合同

- 承接申购单中的采购缺口。
- 确认供应商、采购价格和交付要求。
- 后续可下推提货通知单。

### 提货通知单

- 记录供应商已生产完成、可提货或待提货数量。
- 不增加库存。
- 不触发应付。
- 可分批下推采购入库单。

### 采购入库单

- 记录采购货物实际到库。
- 是库存增加事实。
- 是采购发票和应付核对的重要来源。

### 库存出库单

- 记录销售合同实际出库。
- 是库存扣减事实。
- 是销售发票和应收核对的重要来源。

### 发票和收付款

- 销售发票登记客户正式开票金额。
- 采购发票登记供应商正式开票金额。
- 收款单登记客户实际到账。
- 付款单登记供应商实际付款。

## 讲解重点

- 销售合同是主线。
- 申购单负责锁库存和算采购缺口。
- 提货通知不是库存事实。
- 入库单进入应付链路。
- 出库单进入应收链路。
