# 如何从出库单生成销售发票

本指引用于培训新用户把已确认库存出库单下推为销售发票。示例覆盖查找已确认出库单、核对出库明细、阅读生成确认、生成销售发票草稿、核对来源出库单和销售合同、选择默认入账账户、填写发票号码和开票信息、核对开票金额与出库金额差异、保存确认、查看应收看板，以及确认后续收款单入口。

## 适用场景

- 客户货物已经实际出库，需要登记正式开票金额。
- 财务需要把出库事实转换为正式应收口径。
- 销售发票金额需要和出库金额核对，发现差异后进入发票差异看板处理。
- 后续需要基于销售发票继续登记客户收款。

## 前置条件

- 库存出库单已保存并已确认。
- 出库单已正确关联销售合同。
- 客户、产品、出库数量、销售单价和出库金额已核对。
- 已确认发票号码、发票类型、开票日期和默认入账公司账户。

## 字段填写说明

| 字段 | 是否必填 | 填写方式 | 影响 |
|---|---|---|---|
| 客户 | 必填 | 从出库单自动带出 | 应收、账龄和收款按客户追溯 |
| 来源单号 | 必填 | 由出库单下推自动带出 | 用于核对出库事实和发票金额 |
| 关联合同 | 必填/自动 | 从出库单带出销售合同 | 应收看板、收款和合同履约追溯来源 |
| 币种 | 建议保留 | 从出库单带出 | 保持出库金额和发票金额口径一致 |
| 默认入账账户 | 必填 | 系统按币种带出默认公司账户，可手动调整 | 后续下推收款单时可带出 |
| 发票号码 | 必填 | 填客户正式发票号码 | 销售发票保存确认的必填项 |
| 发票代码 | 按需填写 | 按实际票据填写 | 便于财务查票和归档 |
| 发票类型 | 必填 | 选择增值税专票、普票、形式发票或其他 | 发票分类和后续核对参考 |
| 开票日期 | 必填 | 按实际开票日期填写 | 应收确认和账龄参考 |
| 收票日期 | 可选 | 客户回传或内部归档日期 | 票据流转参考 |
| 不含税金额 | 可选 | 按发票明细填写 | 税额和价税合计核对参考 |
| 税额 | 可选 | 按发票税额填写 | 税务核对参考 |
| 处理状态 | 自动/按需 | 金额一致通常为正常；有差异时选金额异常或处理中 | 发票差异看板跟进状态 |
| 差异处理方式 | 差异时填写 | 客户补款、客户退款、红冲重开等 | 差异闭环依据 |
| 产品明细 | 必填 | 从出库单带出，可按实际发票核对 | 正式应收金额来源 |
| 发票金额 | 自动 | 由产品明细数量和单价计算 | 销售发票正式应收金额 |
| 差异金额 | 自动 | 发票金额减出库金额 | 不为 0 时进入差异处理 |
| 备注 | 按需填写 | 写来源出库单、开票依据和差异说明 | 便于财务、销售和管理层追溯 |
| 保存状态 | 必填 | 草稿 / 已确认 | 已确认后作为正式应收口径 |

关键规则：

```text
出库单 = 实际发货和库存扣减事实
销售发票 = 正式应收口径
收款单 = 客户实际到账和资金流水
```

## 步骤 01：找到已确认出库单

![步骤 01：找到已确认出库单](./screenshots/01-找到已确认出库单.png)

进入“库存管理 > 出库单”，搜索已确认出库单。只有已确认出库单才代表实际发货事实。

## 步骤 02：打开出库单详情

![步骤 02：打开出库单详情](./screenshots/02-打开出库单详情.png)

打开出库单详情，核对客户、来源销售合同、出库状态和出库金额。

## 步骤 03：核对出库明细和金额

![步骤 03：核对出库明细和金额](./screenshots/03-核对出库明细和金额.png)

核对实际出库产品、数量、单价和出库金额。销售发票会以这些出库金额作为核对来源。

## 步骤 04：确认生成销售发票入口

![步骤 04：确认生成销售发票入口](./screenshots/04-确认生成销售发票入口.png)

客户需要开票或公司需要确认正式应收时，点击“生成销售发票”。

## 步骤 05：查看生成销售发票确认

![步骤 05：查看生成销售发票确认](./screenshots/05-查看生成销售发票确认.png)

生成前先确认货物已经实际出库，客户需要开票或公司需要登记正式销售发票。

## 步骤 06：确认销售发票影响

![步骤 06：确认销售发票影响](./screenshots/06-确认销售发票影响.png)

系统会提示：销售发票用于确认正式应收口径，并核对发票金额与出库金额差异。

## 步骤 07：生成销售发票草稿

![步骤 07：生成销售发票草稿](./screenshots/07-生成销售发票草稿.png)

继续生成后，系统生成销售发票草稿，并带出出库单、销售合同、客户和出库产品明细。

## 步骤 08：核对来源出库单和销售合同

![步骤 08：核对来源出库单和销售合同](./screenshots/08-核对来源出库单和销售合同.png)

核对客户、来源出库单、关联合同和币种。销售发票必须能追溯出库单和销售合同。

## 步骤 09：核对默认入账公司账户

![步骤 09：核对默认入账公司账户](./screenshots/09-核对默认入账公司账户.png)

核对默认入账公司账户。销售发票保存确认前必须选择默认入账账户，后续收款单可带出并按实际到账账户调整。

## 步骤 10：填写发票号码和开票信息

![步骤 10：填写发票号码和开票信息](./screenshots/10-填写发票号码和开票信息.png)

填写发票号码、发票代码、发票类型和开票日期。发票号码是销售发票保存确认的必填项。

示例：

| 字段 | 示例 | 说明 |
|---|---|---|
| 发票号码 | INV-XS-20260627-001 | 正式发票号 |
| 发票代码 | XSFP-DEMO-001 | 可按实际票据填写 |
| 发票类型 | 增值税专票 | 按实际发票类型选择 |
| 开票日期 | 2026-06-27 | 按实际开票日期填写 |

## 步骤 11：核对开票明细和发票金额

![步骤 11：核对开票明细和发票金额](./screenshots/11-核对开票明细和发票金额.png)

核对开票产品、数量、单价和价税合计。发票金额会与出库金额对比，差异会进入发票差异看板。

## 步骤 12：核对出库金额和差异金额

![步骤 12：核对出库金额和差异金额](./screenshots/12-核对出库金额和差异金额.png)

核对出库金额、发票金额和差异金额。金额一致时状态为正常；不一致时需要选择差异处理方式并后续闭环。

差异处理参考：

| 情况 | 建议处理 |
|---|---|
| 发票金额大于出库金额 | 核实是否客户补款、补充费用或订单变更 |
| 发票金额小于出库金额 | 核实是否折让、退款、红冲重开或损失处理 |
| 金额录错 | 修正产品行单价、数量或重新开票 |

## 步骤 13：填写备注并保存销售发票

![步骤 13：填写备注并保存销售发票](./screenshots/13-填写备注并保存销售发票.png)

备注中记录来源出库单、开票依据和差异说明。确认无误后点击保存。

备注示例：

```text
由出库单 CK2606274281 生成。客户正式开票金额与出库金额一致，作为正式应收依据。
```

## 步骤 14：保存并确认销售发票

![步骤 14：保存并确认销售发票](./screenshots/14-保存并确认销售发票.png)

资料未齐时可以先保存到草稿；发票号码、默认入账账户、产品明细和金额核对无误后，选择“保存并已确认”。

状态说明：

| 状态 | 适用情况 | 后续影响 |
|---|---|---|
| 保存到草稿 | 发票号码、账户或金额仍需复核 | 不作为正式应收口径 |
| 保存并已确认 | 开票信息和金额已确认 | 作为正式应收口径，可生成收款单 |

## 步骤 15：回到销售发票列表验证

![步骤 15：回到销售发票列表验证](./screenshots/15-回到销售发票列表验证.png)

回到“财务管理 > 销售发票”搜索单号，确认销售发票已经生成并处于已确认状态。

## 步骤 16：查看应收看板正式应收

![步骤 16：查看应收看板正式应收](./screenshots/16-查看应收看板正式应收.png)

应收看板会优先按已确认销售发票作为立账金额；未开票时才按出库单金额参考。

## 步骤 17：确认后续收款单入口

![步骤 17：确认后续收款单入口](./screenshots/17-确认后续收款单入口.png)

销售发票确认后可继续生成收款单。收款单记录客户实际到账，并进入资金流水。

## 常见错误

- 出库单仍是草稿就尝试生成销售发票。
- 销售发票没有关联出库单或销售合同，导致应收追溯断开。
- 未选择默认入账公司账户，导致发票无法保存确认。
- 未填写发票号码。
- 发票产品数量或单价没有和实际发票核对。
- 发票金额和出库金额不一致，但没有填写处理状态和差异处理方式。
- 销售发票只保存到草稿，误以为已经形成正式应收。
- 客户实际到账后忘记继续生成收款单，导致应收长期未结。

## 保存前检查清单

- 出库单状态为已确认。
- 客户、来源出库单和关联合同正确。
- 默认入账公司账户已选择。
- 发票号码、发票类型和开票日期已填写。
- 开票产品、数量、单价和价税合计已核对。
- 出库金额、发票金额和差异金额已核对。
- 如有差异，处理状态和处理方式已填写。
- 备注已写清来源出库单、开票依据和差异说明。
- 如要形成正式应收，保存时选择“保存并已确认”。
- 后续客户到账后，从销售发票生成收款单。
