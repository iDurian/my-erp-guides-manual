# 如何创建财务调整单

本指引用于培训财务用户处理尾差、折让、汇差、坏账、争议扣款等非标准收付款调整事项。示例采用 DPL 销售合同尾差核销，覆盖进入财务调整单列表、新建草稿、选择往来单位、关联合同、填写分摊金额、选择责任公司账户、填写调整原因、保存确认、列表验证、确认不进入资金流水，以及回到单据复核。

## 适用场景

- 客户少付或多付尾差，需要核销余额但没有新的银行收付款。
- 销售或采购发生折让、让利、争议扣款，需要留痕。
- 汇差、手续费差额、四舍五入差额需要调整余额。
- 坏账、无法追回款项或供应商争议损失需要记录。
- 发票差异看板中确认不走退款、补款、费用单或红冲重开，而用财务调整闭环。

## 前置条件

- 已确认调整对象是客户、供应商或其他往来单位。
- 已确认调整归属的销售合同或采购合同。
- 已确认调整金额、币种和原因。
- 已判断是否存在真实银行收付款：如果有真实收付款，应优先使用收款单、付款单、退款单、费用单或报销单。
- 已确认用于责任归集或核对的公司账户。

## 字段填写说明

| 字段 | 是否必填 | 填写方式 | 影响 |
|---|---|---|---|
| 往来单位 | 必填 | 选择客户、供应商或相关往来方 | 决定可分摊的合同范围 |
| 单据日期 | 必填 | 填写调整登记日期 | 影响列表和调整记录日期 |
| 要求日期 | 必填 | 填写调整确认或处理日期 | 便于后续复核 |
| 来源单号 | 可选 | 当前无可用源单时可手工关联说明 | 用于补充来源，不是必填 |
| 关联合同 | 建议必填 | 选择要调整的销售合同或采购合同 | 决定调整归属和追溯口径 |
| 币种 | 必填 | 按合同或调整币种选择 | 决定分摊金额展示币种 |
| 本次分摊金额 | 必填 | 在目标合同行填写调整金额 | 形成调整单金额 |
| 公司账户 | 必填 | 选择责任账户或核对账户 | 财务调整单不进资金流水，但需要账户口径 |
| 备注 | 建议填写 | 写调整类型、合同、金额、原因和是否真实收付 | 便于财务主管和审计追溯 |
| 保存状态 | 必填 | 草稿 / 已确认 | 已确认后成为正式调整记录 |

关键规则：

```text
财务调整单 = 非现金调整留痕
分摊金额 = 调整金额来源
资金流水 = 不包含财务调整单
```

## 步骤 01：进入财务调整单列表

![步骤 01：进入财务调整单列表](./screenshots/01-进入财务调整单列表.png)

进入“财务管理 > 财务调整单”。财务调整单用于处理尾差、折让、汇差、坏账等非收付款调整事项。

## 步骤 02：新建财务调整单草稿

![步骤 02：新建财务调整单草稿](./screenshots/02-新建财务调整单草稿.png)

点击“新增财务调整单”后进入草稿。草稿只保存待核对内容，不作为正式调整事实。

## 步骤 03：填写往来单位和基本信息

![步骤 03：填写往来单位和基本信息](./screenshots/03-填写往来单位和基本信息.png)

选择需要调整余额或留痕的往来单位，并确认日期、负责人、部门和币种。

## 步骤 04：选择需要调整的合同

![步骤 04：选择需要调整的合同](./screenshots/04-选择需要调整的合同.png)

选择关联合同，明确这笔尾差、折让或坏账调整归属到哪张销售或采购合同。

## 步骤 05：查看可分摊目标单据

![步骤 05：查看可分摊目标单据](./screenshots/05-查看可分摊目标单据.png)

选择往来单位后，分摊明细会显示该往来单位名下可调整的客户合同或采购合同。

## 步骤 06：填写本次调整金额

![步骤 06：填写本次调整金额](./screenshots/06-填写本次调整金额.png)

在目标合同行填写本次分摊金额。本例登记 DPL 合同尾差 USD 35.00。

## 步骤 07：核对调整合计

![步骤 07：核对调整合计](./screenshots/07-核对调整合计.png)

核对分摊行数和分摊金额。财务调整单金额来自分摊明细，不来自产品明细。

## 步骤 08：选择责任公司账户

![步骤 08：选择责任公司账户](./screenshots/08-选择责任公司账户.png)

财务调整单必须选择公司账户；没有真实收付时，这里作为责任账户或核对口径，不产生现金流水。

## 步骤 09：填写调整原因备注

![步骤 09：填写调整原因备注](./screenshots/09-填写调整原因备注.png)

备注要写清调整类型、合同编号、金额、原因，以及是否有真实收付款。

备注示例：

```text
DPL 合同 HT2503110001 尾差核销 USD 35.00。该调整仅用于应收余额闭环，没有实际银行收付款，不进入资金流水。
```

## 步骤 10：准备保存财务调整单

![步骤 10：准备保存财务调整单](./screenshots/10-准备保存财务调整单.png)

保存前确认往来单位、关联合同、调整金额、责任账户和调整原因都已填写。

## 步骤 11：选择保存状态

![步骤 11：选择保存状态](./screenshots/11-选择保存状态.png)

资料未核对时保存草稿；调整已确认并可留痕时选择“保存并已确认”。

状态说明：

| 状态 | 适用情况 | 后续影响 |
|---|---|---|
| 保存到草稿 | 调整原因、金额或审批仍需核对 | 不作为正式调整记录 |
| 保存并已确认 | 财务主管或责任人已确认调整 | 形成正式调整记录，但不进入资金流水 |

## 步骤 12：保存并确认财务调整单

![步骤 12：保存并确认财务调整单](./screenshots/12-保存并确认财务调整单.png)

保存并已确认后，财务调整单成为正式调整记录。它不是实际现金事件，因此仍不会进入资金流水。

## 步骤 13：回到调整单列表验证

![步骤 13：回到调整单列表验证](./screenshots/13-回到调整单列表验证.png)

回到财务调整单列表搜索单号，确认状态、往来单位、关联合同和金额。

## 步骤 14：确认不进入资金流水

![步骤 14：确认不进入资金流水](./screenshots/14-确认不进入资金流水.png)

资金流水只展示真实现金事件。财务调整单用于留痕和余额调整，不会出现在现金流水明细中。

## 步骤 15：复核财务调整单明细

![步骤 15：复核财务调整单明细](./screenshots/15-复核财务调整单明细.png)

再次打开调整单，复核合同、分摊金额、责任账户和调整原因，确保后续审计可以追溯。

## 常见错误

- 有真实银行收款或付款，却误用财务调整单，导致资金流水缺失。
- 没有选择往来单位，分摊明细无法显示目标合同。
- 没有关联合同，后续无法判断调整归属。
- 分摊金额填错币种或金额，导致余额核对异常。
- 备注只写“调整”或“尾差”，没有说明原因、合同和是否真实收付。
- 把费用类真实支出写成财务调整单，应改用费用单或报销单。
- 把客户退款或供应商退款写成财务调整单，应改用对应退款单。
- 以为财务调整单会进入资金流水，实际它只做调整留痕。

## 保存前检查清单

- 往来单位已选择正确。
- 关联合同已选择，调整归属明确。
- 本次分摊金额与审批或核销依据一致。
- 币种与合同或调整依据一致。
- 公司账户已选择为责任账户或核对账户。
- 备注写清调整类型、合同编号、金额、原因和是否真实收付。
- 已判断没有真实现金流；若有真实收付，应改用收付款、退款、费用或报销单。
- 如调整已确认，保存时选择“保存并已确认”。
