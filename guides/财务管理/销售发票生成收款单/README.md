# 如何从销售发票生成收款单

本指引用于培训新用户把已确认销售发票下推为收款单。示例覆盖查找已确认销售发票、核对发票信息和正式应收、阅读生成确认、生成收款单草稿、核对来源销售发票和销售合同、选择实际入账公司账户、核对收款明细和到账金额、保存确认、查看应收看板待收减少，以及查看资金流水入账记录。

## 适用场景

- 客户已实际付款到账，需要登记收款。
- 财务需要冲减销售发票形成的正式应收。
- 一张销售发票可能分多次收款，需要按每次实际到账分别登记。
- 需要让资金流水按公司账户记录现金流入。

## 前置条件

- 销售发票已保存并已确认。
- 销售发票已正确关联销售合同。
- 已确认客户实际到账金额、到账日期和入账公司账户。
- 如果是部分收款，已明确本次收款金额和剩余待收金额。

## 字段填写说明

| 字段 | 是否必填 | 填写方式 | 影响 |
|---|---|---|---|
| 客户 | 必填 | 从销售发票自动带出 | 应收、账龄和资金流水按客户追溯 |
| 来源单号 | 必填 | 由销售发票下推自动带出 | 用于追溯正式应收来源 |
| 关联合同 | 必填/自动 | 从销售发票带出销售合同 | 应收看板、收款和合同履约追溯来源 |
| 币种 | 建议保留 | 从销售发票带出 | 保持应收和收款口径一致 |
| 入账账户 | 必填 | 系统带出默认账户，可按实际到账账户调整 | 保存确认后进入资金流水 |
| 产品 / 费用金额明细 | 必填 | 从销售发票带出 | 作为本次收款金额明细 |
| 数量 | 必填 | 全额收款可保持发票数量；部分收款可按金额折算 | 参与本次收款金额计算 |
| 收款单价 | 必填 | 全额收款可保持发票单价；部分收款可按实际到账调整 | 参与本次收款金额计算 |
| 价税合计 | 自动 | 由数量、收款单价和税率计算 | 本次实际到账金额 |
| 备注 | 按需填写 | 写来源销售发票、到账金额、到账账户和差异说明 | 便于财务、销售和管理层追溯 |
| 保存状态 | 必填 | 草稿 / 已确认 | 已确认后冲减应收并进入资金流水 |

关键规则：

```text
销售发票 = 正式应收口径
收款单 = 客户实际到账事实
资金流水 = 只统计已确认的真实现金事件
```

## 步骤 01：找到已确认销售发票

![步骤 01：找到已确认销售发票](./screenshots/01-找到已确认销售发票.png)

进入“财务管理 > 销售发票”，搜索已确认销售发票。收款单应从正式应收来源继续生成。

## 步骤 02：打开销售发票详情

![步骤 02：打开销售发票详情](./screenshots/02-打开销售发票详情.png)

打开销售发票详情，核对客户、来源出库单、销售合同、发票状态和正式应收金额。

## 步骤 03：核对发票信息和正式应收

![步骤 03：核对发票信息和正式应收](./screenshots/03-核对发票信息和正式应收.png)

核对发票号码、开票日期、发票金额和差异金额。收款单会按实际到账金额冲减这笔应收。

## 步骤 04：核对开票明细和金额

![步骤 04：核对开票明细和金额](./screenshots/04-核对开票明细和金额.png)

核对开票产品、数量、收款单价和价税合计。生成收款单后可按实际到账金额调整。

## 步骤 05：确认生成收款单入口

![步骤 05：确认生成收款单入口](./screenshots/05-确认生成收款单入口.png)

客户回款已经到账，或需要根据销售发票登记本次收款时，点击“生成收款单”。

## 步骤 06：查看生成收款单确认

![步骤 06：查看生成收款单确认](./screenshots/06-查看生成收款单确认.png)

生成前先确认客户回款已经到账，或者本次需要根据销售发票登记实际收款。

## 步骤 07：确认收款单资金影响

![步骤 07：确认收款单资金影响](./screenshots/07-确认收款单资金影响.png)

系统会提示：收款单需要按实际到账金额和入账公司账户确认，保存后进入资金流水。

## 步骤 08：生成收款单草稿

![步骤 08：生成收款单草稿](./screenshots/08-生成收款单草稿.png)

继续生成后，系统生成收款单草稿，并带出销售发票、销售合同、客户和产品金额明细。

## 步骤 09：核对来源销售发票和销售合同

![步骤 09：核对来源销售发票和销售合同](./screenshots/09-核对来源销售发票和销售合同.png)

核对客户、来源销售发票、关联合同和币种。收款单必须能追溯正式应收来源。

## 步骤 10：核对实际入账公司账户

![步骤 10：核对实际入账公司账户](./screenshots/10-核对实际入账公司账户.png)

核对实际入账公司账户。收款单保存确认后会按这个账户进入资金流水。

## 步骤 11：核对收款明细和到账金额

![步骤 11：核对收款明细和到账金额](./screenshots/11-核对收款明细和到账金额.png)

核对收款产品、数量、收款单价和到账金额。本次全额到账时保持与销售发票金额一致。

## 步骤 12：按实际到账调整收款金额

![步骤 12：按实际到账调整收款金额](./screenshots/12-按实际到账调整收款金额.png)

如果客户部分付款，可以把数量或收款单价改成本次实际到账金额；全额到账则保持发票金额。

示例：

| 情况 | 填写方式 | 影响 |
|---|---|---|
| 全额到账 | 保持发票数量和收款单价 | 应收看板待收归零 |
| 部分到账 | 按本次实际到账金额调整数量或单价 | 应收看板保留剩余待收 |
| 多账户到账 | 按实际账户分别登记收款单 | 资金流水按账户拆分 |

## 步骤 13：填写备注并保存收款单

![步骤 13：填写备注并保存收款单](./screenshots/13-填写备注并保存收款单.png)

备注中记录来源销售发票、到账金额、到账账户和差异说明。确认无误后点击保存。

备注示例：

```text
由销售发票 XSFP2606271234 生成。本次客户全额到账 USD 3,750.00，入美元主收款账户。
```

## 步骤 14：保存并确认收款单

![步骤 14：保存并确认收款单](./screenshots/14-保存并确认收款单.png)

如果到账金额或账户仍需复核，可以先保存到草稿；确认实际到账后，选择“保存并已确认”。

状态说明：

| 状态 | 适用情况 | 后续影响 |
|---|---|---|
| 保存到草稿 | 到账金额、账户或备注仍需复核 | 不冲减应收，不进入资金流水 |
| 保存并已确认 | 实际到账已经确认 | 冲减应收，并进入资金流水 |

## 步骤 15：回到收款单列表验证

![步骤 15：回到收款单列表验证](./screenshots/15-回到收款单列表验证.png)

回到“财务管理 > 收款单”搜索单号，确认收款单已经生成并处于已确认状态。

## 步骤 16：查看应收看板待收减少

![步骤 16：查看应收看板待收减少](./screenshots/16-查看应收看板待收减少.png)

应收看板会把收款单计入累计收款；全额收款后待收余额归零。

## 步骤 17：查看资金流水入账记录

![步骤 17：查看资金流水入账记录](./screenshots/17-查看资金流水入账记录.png)

资金流水会记录已确认收款单的现金流入，可按公司账户和日期核对到账。

## 常见错误

- 销售发票仍是草稿就尝试生成收款单。
- 收款单没有关联销售合同，导致应收和合同追溯断开。
- 未选择实际入账公司账户，导致收款单无法保存确认。
- 客户只部分付款，却按全额发票登记收款。
- 多账户到账只登记在一个账户，导致资金流水账户不准。
- 收款单只保存到草稿，误以为已经冲减应收。
- 收款金额和银行实际到账金额不一致，但备注没有说明。

## 保存前检查清单

- 销售发票状态为已确认。
- 客户、来源销售发票和关联合同正确。
- 实际入账公司账户已选择。
- 收款产品、数量、收款单价和价税合计已核对。
- 本次收款金额与银行实际到账金额一致。
- 如果是部分收款，剩余待收金额已确认。
- 备注已写清来源销售发票、到账金额、到账账户和差异说明。
- 如要冲减应收并进入资金流水，保存时选择“保存并已确认”。
