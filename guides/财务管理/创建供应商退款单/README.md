# 如何创建供应商退款单

本指引用于培训新用户登记“供应商退回我司的实际到账”。示例采用原付款单超付款后供应商退回部分款项的场景，覆盖查找原付款单、新建供应商退款单、选择来源、核对供应商和采购合同、选择实际入账账户、填写理论应退金额、本次实退金额和差异原因、保存确认，以及查看资金流水入账记录。

## 适用场景

- 供应商退回超付款。
- 供应商退回采购发票差异款、折让款或质量扣款。
- 供应商退回费用、版辊款或其他垫付款。
- 财务需要登记真实现金流入，并让退款原因可追溯到原付款单、采购发票或费用单。

## 前置条件

- 已确认原付款单、采购发票或费用单。
- 已和供应商对清理论应退金额、本次实际到账金额和未退差额。
- 已确认实际入账日期、入账公司账户和币种。
- 如果本次实退金额小于理论应退金额，已明确扣费或差异原因。

## 字段填写说明

| 字段 | 是否必填 | 填写方式 | 影响 |
|---|---|---|---|
| 供应商/货代 | 必填 | 从来源付款单、采购发票或费用单带出 | 资金流水、应付和供应商追溯依据 |
| 来源单号 | 必填 | 选择原付款单、采购发票或费用单 | 保存确认前必须关联原业务来源 |
| 关联合同 | 建议必填 | 从来源自动带出采购合同 | 便于采购、财务和管理层按合同追溯 |
| 币种 | 建议保留 | 从来源带出，或按实际到账币种选择 | 决定金额展示和资金流水币种 |
| 入账账户 | 必填 | 选择实际到账的公司账户 | 保存确认后按该账户进入资金流水 |
| 理论应退金额 | 建议填写 | 按对账结果填写供应商应退金额 | 用于和本次实退金额比较 |
| 本次实退金额 | 自动 | 由退款明细金额汇总 | 本次真实现金流入金额 |
| 扣费/差异金额 | 自动 | 理论应退金额减本次实退金额 | 大于 0 时必须说明原因 |
| 扣费原因 | 差异时必填 | 写银行手续费、质量争议、供应商扣费等 | 作为差异闭环依据 |
| 费用类别 | 必填 | 通常选择“供应商退款”，版辊返还可选择“版辊费” | 影响分类核对 |
| 摘要 | 必填 | 写供应商退款和来源单号 | 资金流水和列表中快速识别 |
| 单价/金额 | 必填 | 填本次实际到账金额 | 参与现金流入金额计算 |
| 备注 | 建议填写 | 写来源、理论应退、实际到账、差异原因 | 便于财务、采购和审计追溯 |
| 保存状态 | 必填 | 草稿 / 已确认 | 已确认后进入资金流水 |

关键规则：

```text
原付款单 / 采购发票 / 费用单 = 退款原因来源
供应商退款单 = 供应商实际退回我司的现金流入
理论应退 > 本次实退 = 必须填写扣费或差异原因
```

## 步骤 01：找到原付款单

![步骤 01：找到原付款单](./screenshots/01-找到原付款单.png)

进入“财务管理 > 付款单”，找到发生超付款或差异款的原付款单。

## 步骤 02：打开付款单详情

![步骤 02：打开付款单详情](./screenshots/02-打开付款单详情.png)

打开付款单详情，核对供应商、采购合同、付款状态和已付金额。

## 步骤 03：确认供应商应退来源

![步骤 03：确认供应商应退来源](./screenshots/03-确认供应商应退来源.png)

确认本次供应商退款来自原付款单，例如超付款、质量扣款、费用差异或版辊返还。

## 步骤 04：进入供应商退款单列表

![步骤 04：进入供应商退款单列表](./screenshots/04-进入供应商退款单列表.png)

进入“财务管理 > 供应商退款单”。供应商退款单用于登记供应商退回我司的实际到账。

## 步骤 05：新建供应商退款单草稿

![步骤 05：新建供应商退款单草稿](./screenshots/05-新建供应商退款单草稿.png)

点击“新增供应商退款单”后进入草稿。草稿不会进入资金流水。

## 步骤 06：选择来源付款单

![步骤 06：选择来源付款单](./screenshots/06-选择来源付款单.png)

在来源单号选择原付款单，系统会带出供应商、采购合同、币种和退款明细。

## 步骤 07：核对供应商来源和采购合同

![步骤 07：核对供应商来源和采购合同](./screenshots/07-核对供应商来源和采购合同.png)

核对供应商、来源付款单和采购合同，保证退款可以追溯到原应付链路。

## 步骤 08：选择实际入账公司账户

![步骤 08：选择实际入账公司账户](./screenshots/08-选择实际入账公司账户.png)

供应商退款单必须选择实际入账公司账户，保存确认后资金流水按这个账户记录现金流入。

## 步骤 09：核对供应商退款信息

![步骤 09：核对供应商退款信息](./screenshots/09-核对供应商退款信息.png)

供应商退款信息用于对比理论应退、本次实退和未退差额；未退差额必须说明原因。

## 步骤 10：填写理论应退金额

![步骤 10：填写理论应退金额](./screenshots/10-填写理论应退金额.png)

按对账结果填写理论应退金额。例如原付款单多付 CNY 200.00，就把理论应退金额填为 200。

## 步骤 11：填写本次实退金额和差异原因

![步骤 11：填写本次实退金额和差异原因](./screenshots/11-填写本次实退金额和差异原因.png)

本次实退金额按银行实际到账填写；理论应退大于实退时，需要填写扣费或差异原因。

示例：

| 情况 | 填写方式 | 影响 |
|---|---|---|
| 全额退回 | 理论应退金额 = 本次实退金额 | 无扣费/差异金额 |
| 扣手续费后退回 | 本次实退金额小于理论应退金额，填写扣费原因 | 差异金额保留为后续核对依据 |
| 只退部分金额 | 本次实退金额按实际到账填写 | 备注说明剩余金额后续处理 |

## 步骤 12：复核实退差异和原因

![步骤 12：复核实退差异和原因](./screenshots/12-复核实退差异和原因.png)

复核本次实退金额、扣费/差异金额和原因。差异后续可通过财务调整单闭环。

## 步骤 13：填写备注并准备保存

![步骤 13：填写备注并准备保存](./screenshots/13-填写备注并准备保存.png)

备注写清来源付款单、理论应退、实际到账、差异原因和入账账户，便于财务追溯。

备注示例：

```text
关联付款单 FK2606289201。理论应退 CNY 200.00，供应商实际退回 CNY 180.00，差额为银行手续费。
```

## 步骤 14：选择保存状态

![步骤 14：选择保存状态](./screenshots/14-选择保存状态.png)

金额或到账账户未确认时保存到草稿；供应商退款已到账时选择“保存并已确认”。

状态说明：

| 状态 | 适用情况 | 后续影响 |
|---|---|---|
| 保存到草稿 | 到账金额、账户或差异原因仍需复核 | 不进入资金流水 |
| 保存并已确认 | 供应商退款已经实际到账 | 进入资金流水，作为现金流入 |

## 步骤 15：保存并确认供应商退款单

![步骤 15：保存并确认供应商退款单](./screenshots/15-保存并确认供应商退款单.png)

保存并已确认后，供应商退款单成为实际现金流入，会进入资金流水。

## 步骤 16：回到供应商退款单列表验证

![步骤 16：回到供应商退款单列表验证](./screenshots/16-回到供应商退款单列表验证.png)

回到供应商退款单列表搜索单号，确认退款单已经生成并处于已确认状态。

## 步骤 17：查看资金流水入账记录

![步骤 17：查看资金流水入账记录](./screenshots/17-查看资金流水入账记录.png)

资金流水会记录已确认供应商退款单的现金流入，可按账户和日期核对到账。

## 常见错误

- 没有关联原付款单、采购发票或费用单，导致无法说明退款来源。
- 未选择实际入账公司账户，导致无法保存确认。
- 理论应退金额填成原付款单全额，而不是本次供应商应退金额。
- 本次实退金额和银行实际到账不一致。
- 理论应退大于本次实退，但没有填写扣费或差异原因。
- 退款币种和银行实际到账币种不一致。
- 供应商退款单只保存到草稿，误以为已经进入资金流水。
- 差异金额长期未处理，没有用财务调整单或后续退款闭环。

## 保存前检查清单

- 供应商、来源单号和采购合同正确。
- 来源类型为付款单、采购发票或费用单。
- 实际入账公司账户已选择。
- 理论应退金额已按对账结果填写。
- 本次实退金额与银行实际到账一致。
- 如有扣费/差异金额，扣费原因已填写。
- 备注已写清来源、理论应退、实际到账和差异原因。
- 如要形成真实现金流入，保存时选择“保存并已确认”。
