# 如何创建客户退款单

本指引用于培训新用户登记“退还客户的实际出账”。示例采用直接退款型客诉处理单作为来源，覆盖查找客诉、核对退款原因、新建客户退款单、选择来源、核对客户和销售合同、选择实际出账账户、填写退款金额、保存确认、验证客诉已退款，以及查看资金流水出账记录。

## 适用场景

- 客户客诉确认需要直接退款。
- 客户多付款、重复付款或金额有误，需要退回客户。
- 版辊费达到返还条件，需要退还客户。
- 财务需要登记真实现金流出，并让退款原因可追溯到合同、收款单、客诉或版辊记录。

## 前置条件

- 已确认退款原因、退款金额、退款日期和实际出账账户。
- 如果来源是客诉，客诉处理单应为“直接退款”且仍未处理。
- 如果来源是销售合同或收款单，需要确认退款与该客户、合同和回款记录匹配。
- 如为部分退款，需明确剩余待处理金额和后续处理方式。

## 字段填写说明

| 字段 | 是否必填 | 填写方式 | 影响 |
|---|---|---|---|
| 客户 | 必填 | 从来源客诉/合同/收款单带出，也可手动选择 | 资金流水、客诉闭环和客户追溯依据 |
| 来源单号 | 建议必填 | 选择客诉处理单、销售合同或收款单 | 用于说明为什么退款，避免孤立出账 |
| 关联合同 | 建议必填 | 从来源自动带出销售合同，可手动核对 | 方便销售、财务和管理层按合同追溯 |
| 币种 | 建议保留 | 从来源带出，或按实际退款币种选择 | 决定金额展示和资金流水币种 |
| 出账账户 | 必填 | 选择实际付款的公司账户 | 保存确认后按该账户进入资金流水 |
| 费用类别 | 必填 | 客诉退款可用“退款扣费”，版辊返还可用“版辊费” | 影响后续分类核对 |
| 摘要 | 必填 | 写退款原因或来源单号 | 资金流水和列表中快速识别 |
| 数量 | 必填 | 通常为 1；拆分退款可按实际业务拆行 | 参与退款金额计算 |
| 单价/金额 | 必填 | 填本次实际退给客户的金额 | 本次现金流出金额 |
| 备注 | 建议填写 | 写来源、退款原因、审批依据和出账账户 | 便于销售、财务、审计追溯 |
| 保存状态 | 必填 | 草稿 / 已确认 | 已确认后进入资金流水；客诉来源会自动闭环 |

关键规则：

```text
客诉处理单 = 退款原因和处理安排
客户退款单 = 实际退给客户的现金流出
资金流水 = 只统计已确认的真实现金事件
```

## 步骤 01：找到直接退款型客诉处理单

![步骤 01：找到直接退款型客诉处理单](./screenshots/01-找到直接退款型客诉处理单.png)

进入“销售管理 > 客诉处理”，搜索处理类型为直接退款的客诉处理单。

## 步骤 02：打开客诉处理单详情

![步骤 02：打开客诉处理单详情](./screenshots/02-打开客诉处理单详情.png)

打开客诉处理单，核对客户、来源销售合同、客诉状态和客诉金额。

## 步骤 03：核对客诉金额和退款去向

![步骤 03：核对客诉金额和退款去向](./screenshots/03-核对客诉金额和退款去向.png)

处理类型为“直接退款”时，后续需要到财务管理新建客户退款单，并在来源单号中选择这张客诉单。

## 步骤 04：进入客户退款单列表

![步骤 04：进入客户退款单列表](./screenshots/04-进入客户退款单列表.png)

进入“财务管理 > 客户退款单”。客户退款单用于登记退还客户的实际出账。

## 步骤 05：新建客户退款单草稿

![步骤 05：新建客户退款单草稿](./screenshots/05-新建客户退款单草稿.png)

点击“新增客户退款单”后进入草稿。草稿不会冲减回款，也不会进入资金流水。

## 步骤 06：选择来源客诉处理单

![步骤 06：选择来源客诉处理单](./screenshots/06-选择来源客诉处理单.png)

在来源单号选择直接退款型客诉处理单，系统会带出客户、销售合同、币种和退款明细。

## 步骤 07：核对客户来源和销售合同

![步骤 07：核对客户来源和销售合同](./screenshots/07-核对客户来源和销售合同.png)

核对客户、来源客诉单和关联合同。来源关系正确，后续客诉状态才能自动变为已退款。

## 步骤 08：选择实际出账公司账户

![步骤 08：选择实际出账公司账户](./screenshots/08-选择实际出账公司账户.png)

客户退款单必须选择实际出账公司账户，保存确认后资金流水按这个账户记录现金流出。

## 步骤 09：核对退款明细和金额

![步骤 09：核对退款明细和金额](./screenshots/09-核对退款明细和金额.png)

核对费用类别、摘要、数量、单价和合计金额。客诉来源会自动带出退款金额。

## 步骤 10：按实际出账调整退款金额

![步骤 10：按实际出账调整退款金额](./screenshots/10-按实际出账调整退款金额.png)

如果实际只退部分金额，可按银行出账调整单价或数量；全额退款则保持来源金额。

示例：

| 情况 | 填写方式 | 影响 |
|---|---|---|
| 全额退款 | 保持来源金额 | 客诉直接闭环，资金流水记录全额出账 |
| 部分退款 | 修改单价/金额为本次实际出账 | 备注中说明剩余金额如何处理 |
| 多账户退款 | 按账户分别登记客户退款单 | 资金流水按账户拆分 |

## 步骤 11：填写备注并准备保存

![步骤 11：填写备注并准备保存](./screenshots/11-填写备注并准备保存.png)

备注写清来源客诉单、退款原因、实际退款金额和出账账户，便于财务与销售追溯。

备注示例：

```text
关联客诉处理单 KS2606289101。客户直接退款 USD 180.00，按实际出账账户登记。
```

## 步骤 12：选择保存状态

![步骤 12：选择保存状态](./screenshots/12-选择保存状态.png)

金额或账户未确认时保存到草稿；实际退款已出账时选择“保存并已确认”。

状态说明：

| 状态 | 适用情况 | 后续影响 |
|---|---|---|
| 保存到草稿 | 金额、账户或审批仍需确认 | 不进入资金流水，不关闭客诉 |
| 保存并已确认 | 实际退款已经出账 | 进入资金流水；客诉来源自动显示已退款 |

## 步骤 13：保存并确认客户退款单

![步骤 13：保存并确认客户退款单](./screenshots/13-保存并确认客户退款单.png)

保存并已确认后，客户退款单成为实际出账事实，会进入资金流水并用于客诉闭环。

## 步骤 14：回到客户退款单列表验证

![步骤 14：回到客户退款单列表验证](./screenshots/14-回到客户退款单列表验证.png)

回到客户退款单列表搜索单号，确认客户退款单已生成并处于已确认状态。

## 步骤 15：验证客诉状态已退款

![步骤 15：验证客诉状态已退款](./screenshots/15-验证客诉状态已退款.png)

客户退款单确认后，关联的客诉处理单会自动显示已退款，并记录处理去向。

## 步骤 16：查看资金流水出账记录

![步骤 16：查看资金流水出账记录](./screenshots/16-查看资金流水出账记录.png)

资金流水会记录已确认客户退款单的现金流出，可按账户和日期核对退款出账。

## 步骤 17：查看应收看板退款影响

![步骤 17：查看应收看板退款影响](./screenshots/17-查看应收看板退款影响.png)

客户退款属于现金流出，会影响合同回款口径。直接客诉退款如果没有对应出库或销售发票，应收看板可能没有立账行；此时应以客诉状态和资金流水作为主要核对依据。

## 常见错误

- 客诉处理类型不是“直接退款”，却创建客户退款单。
- 客户退款单没有选择来源单号，后续无法解释退款原因。
- 未选择实际出账公司账户，导致无法保存确认。
- 实际只退部分金额，却按客诉全额登记。
- 退款币种和银行实际出账币种不一致。
- 客户退款单只保存到草稿，误以为已经进入资金流水。
- 多账户退款只登记在一个账户，导致资金流水账户不准。
- 客诉退款后没有回到客诉处理单确认状态是否已退款。

## 保存前检查清单

- 客户、来源单号和关联合同正确。
- 来源客诉为直接退款，且仍未处理。
- 实际出账公司账户已选择。
- 费用类别、摘要、数量、单价/金额已核对。
- 本次退款金额与银行实际出账金额一致。
- 如果是部分退款，备注已说明剩余金额处理方式。
- 备注已写清退款原因、来源单号和出账依据。
- 如要形成真实现金流出，保存时选择“保存并已确认”。
