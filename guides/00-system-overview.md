# 系统定位与整体认知

本模块用于帮助新用户先理解系统解决什么问题，以及日常使用时应该围绕哪条业务主线展开。

## 核心定位

- 这是一套外贸 ERP，用于管理从报价、销售合同、申购、采购、入库、出库、发票、收付款到附件归档的完整业务链。
- 系统以销售合同为核心追溯对象，后续采购、库存、财务和文档都围绕合同展开。
- 使用目标是减少重复录入、明确下一步处理事项，并让管理层和财务能看到货、钱、单据的风险。

## 新用户需要建立的基本认知

- 业务不是孤立录单，而是从上游单据下推到下游单据。
- 销售合同是主线，采购、库存、财务和文档都要能追溯回销售合同。
- 入库和出库是库存变化的事实来源。
- 收款和付款是资金流水的事实来源。
- 工作台和报表用于发现风险，各模块单据页用于处理具体业务。

## 讲解建议

开场时先用一两句话说明系统价值，再展示主菜单结构。不要一开始就逐项解释所有菜单，否则新用户容易只记住入口，理解不了业务链路。
