# 实际业务流程教程索引

本目录按外贸公司真实业务流程拆分教程。每个流程从某个角色的业务需求出发，截图顶部用红色提示明确“本步点哪里、填哪里、核对哪里”。

## 建议学习顺序

- [流程 01：客户要报价，销售如何创建正式报价单](./01-customer-quotation/README.md)：客户发来询价，销售需要把客户、产品、数量、币种、单价和有效期沉淀成可追溯报价。
- [流程 02：客户确定下单，销售如何转销售合同并下推申购](./02-order-confirmation-requisition/README.md)：客户接受报价后，销售要把报价转成正式销售合同，并让采购和仓库看到真实需求。
- [流程 03：采购收到申购缺口，如何给供应商下单并登记生产完成](./03-purchase-order-production/README.md)：销售合同产生采购缺口后，采购要按缺口给供应商下采购合同，并在供应商生产完成时登记提货通知。
- [流程 04：供应商发货后，仓管如何入库并给客户出库](./04-inbound-outbound/README.md)：货物实际移动时，仓管要记录真实入库和真实出库，避免把 ready 状态误认为库存。
- [流程 05：客户出货后，财务如何开销售发票并登记收款](./05-sales-invoice-receipt/README.md)：出库完成后，财务要确认正式应收，客户到账后登记真实现金流入。
- [流程 06：供应商开票后，财务如何登记采购发票并付款](./06-purchase-invoice-payment/README.md)：供应商开票并要求付款时，财务要从入库事实确认正式应付，再按银行实际付款登记现金流出。
- [流程 07：客户投诉要求退款，销售和财务如何闭环](./07-complaint-refund/README.md)：客户提出质量、数量、包装或交期问题并要求退款时，销售先登记客诉，财务再登记真实退款出账。
- [流程 08：客户投诉改为下一单打折，销售和财务如何留痕](./08-complaint-next-order-discount/README.md)：客户不要求立即退款，而要求下一单折扣时，销售要记录原客诉，下一单价格要体现抵扣，财务要解释发票差异或调整。
- [流程 09：客户达标后，如何退版辊费并追踪供应商返还](./09-plate-roll-refund/README.md)：产品涉及版辊费时，公司要同时追踪客户收费、供应商付款、达标返还和实际现金流。
- [流程 10：财务如何跟踪发票差异、账龄和资金流水](./10-finance-tracking/README.md)：当发票、收付款、退款、费用或调整发生后，财务和管理层要能解释每笔差异和余额。

## 角色参与关系

| 角色 | 主要流程 | 不应直接处理 |
|---|---|---|
| 销售/业务员 | 报价、销售合同、申购发起、客诉、下一单折扣、版辊客户侧返还 | 采购价格、供应商付款、资金流水全表 |
| 采购员 | 申购缺口、采购合同、供应商 ready、供应商版辊返还 | 客户销售价、客户收款、客户应收 |
| 仓管员 | 入库、出库、库存数量事实 | 销售价格、采购价格、发票和收付款 |
| 财务 | 销售发票、采购发票、收款、付款、退款、调整、资金流水 | 修改销售/采购/库存业务明细 |
| 管理层 | 履约、账龄、资金、差异和异常闭环 | 日常录入和绕过责任人直接改单 |
| 只读审计 | 只读核对合同、附件、单据、报表和资金记录 | 新增、编辑、下推、作废 |

## 端到端流程图

```text
客户询价
  -> 报价单
  -> 销售合同
  -> 申购单
  -> 采购合同
  -> 提货通知 / 供应商生产完成
  -> 采购入库单
  -> 库存出库单
  -> 销售发票 / 收款单
  -> 采购发票 / 付款单
  -> 客诉、版辊费、发票差异、资金流水跟踪
```

## 使用说明

- 培训新员工时，不要从单据字段开始讲，先选择他所在角色当天要完成的流程。
- 每张截图的红色顶部提示是本步操作目标；页面正文只补充业务原因和交接关系。
- 涉及真实收付、退款、费用、报销的流程必须由财务完成，并在资金流水中核对。
- 没有真实现金流的折让、尾差、汇差或坏账，用财务调整单留痕，不进入资金流水。
